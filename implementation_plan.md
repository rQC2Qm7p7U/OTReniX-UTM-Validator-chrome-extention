# План реализации v3.5: Интеграция с AI Patch Assistant (Gemini / Prompt API) — Улучшенная Версия

Этот документ описывает технический дизайн и пошаговый план внедрения интеллектуального помощника (**AI Patch Assistant**) в расширение **Dynamic UTM & Lead Source Validator** с применением лучших практик веб-разработки и безопасности расширений Chrome.

---

## Goal Description

Мы добавили AI Patch Assistant, который будет анализировать структуру конкретной формы и генерировать точный JavaScript-патч. 
Система будет поддерживать три сценария:
1. **Копирование промпта (Manual LLM):** Экспорт контекстного промпта с DOM-структурой формы для ручной вставки во внешние чаты.
2. **Локальный ИИ в браузере (Chrome Prompt API / Gemini Nano):** Вызов `window.ai` для генерации кода без сетевых запросов и API-ключей.
3. **Gemini API Key:** Запрос к Google Gemini API через API-ключ в настройках расширения.

---

## 💎 Лучшие практики, добавленные в план (Улучшения)

Для создания стабильного и безопасного enterprise-grade функционала мы внедряем следующие улучшения:

### 1. Безопасность и изоляция учетных данных (Credential Security)
* **Локальное хранение:** API-ключ Gemini будет храниться исключительно в `chrome.storage.local`. В отличие от `chrome.storage.sync`, локальное хранилище никогда не синхронизируется через облако Google и не передается между устройствами пользователя, что снижает риск утечки ключа.
* **Маскирование ключа:** Поле ввода в настройках будет маскироваться (`type="password"`), а в UI ключ будет отображаться в сокращенном виде (например, `AIzaSy...4aBc`).

### 2. Динамическая минимизация контекста (Token Optimization)
* Вместо передачи всех возможных UTM-меток, расширение будет на лету вычислять список параметров, **недостающих именно для этой формы**. 
* Промпт будет содержать точечную инструкцию: *«Сгенерируй скрипт для добавления полей [X, Y, Z], так как поля [A, B] уже присутствуют в форме»*. Это уменьшает размер промпта, снижает стоимость запросов к API и повышает качество генерируемого кода.

### 3. Инструкции по настройке Chrome Prompt API (User Enablement)
* Встроенный Prompt API (`window.ai`) в Chrome по умолчанию скрыт за флагами. Если пользователь пытается запустить локальную генерацию, но `window.ai` недоступен, расширение покажет не просто ошибку, а пошаговую инструкцию по активации:
  1. Откройте `chrome://flags/#optimization-guide-on-device-model` -> выберите **Enabled BypassPrefRequirement**.
  2. Откройте `chrome://flags/#prompt-api-for-gemini-nano` -> выберите **Enabled**.
  3. Перезапустите Chrome и подождите скачивание модели в `chrome://components` (модель `Optimization Guide On Device Model`).

### 4. Улучшенный UX загрузки и обработки ошибок (Robust Error Recovery)
* **Skeleton Loader:** Во время генерации кода область вывода будет отображать анимированный скелетон-загрузчик вместо статичного спиннера, показывая структуру будущего ответа.
* **Диагностика ошибок:** Обработка HTTP-статусов при работе с API:
  - `401 Unauthorized` -> сообщение «Неверный API-ключ. Проверьте настройки расширения».
  - `429 Too Many Requests` -> сообщение «Превышен лимит запросов. Пожалуйста, подождите или используйте локальный ИИ».
  - `Offline` -> проверка сетевого соединения.

### 5. Копирование кода в один клик с обратной связью (Micro-interactions)
* Кнопки копирования кода и промпта будут иметь интерактивную анимацию: при нажатии иконка копирования временно меняется на зеленую галочку, а текст кнопки — на "Copied!", что дает четкую обратную связь.
